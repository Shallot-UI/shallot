# Color Palettes

In Shallot, color palettes are stored in the colors object within the theme. These palettes are used to provide a consistent and visually pleasing set of colors for your application.

## The Shading Palette

Each color palette is represented by a color name and a set of shade values ranging from 100 to 900 in increments of `100`. (There are two included values of 50 and 950 at each end of the scale) The color names in the default theme are `'Primary'`, `'Success'`, `'Warning'`, `'Danger'`, and `'Shading'`.

Below is Sallot's default shading palette. It will be used if no other shading palette is provided in the theme.

<Canvas>
  <ShallotProvider>
    <Row>
      {[
        'Shading.50',
        'Shading.100',
        'Shading.200',
        'Shading.300',
        'Shading.400',
        'Shading.500',
        'Shading.600',
        'Shading.700',
        'Shading.800',
        'Shading.900',
        'Shading.950',
      ].map((color) => (
        <Box
          key={color}
          shallot={{
            backgroundColor: getColorShade(color),
            height: getUnits(5),
            display: 'flex',
            flexGrow: 1,
            flexShrink: 1,
          }}
        />
      ))}
    </Row>
  </ShallotProvider>
</Canvas>

## Contextual Color Palettes

The contextual color palettes are used to communicate different states and actions. Each has its own meaning and should be used consistently throughout your application. It's important to note that these colors will be used by some Shallot components by default, so they should not be overridden in ways that don't align with their intended meaning.

### Primary

The primary color palette is used to communicate that an element is the primary action or the primary element on the page. For example, it is used to style the primary button, primary text, and other elements that require a consistent color.
